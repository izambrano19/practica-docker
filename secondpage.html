DOCKER



¿QUÉ ES DOCKER?


Es una plataforma para contenedores. Se instala en el servidor en el cual podrás ejecutar los contenedores. Podrás crear, iniciar y detener contenedores.
Básicamente sirve para desarrollar y ejecutar aplicaciones dentro de contenedores.



PRINCIPALES CARACTERÍSTICAS


  • Fiable.
  • Es capaz de desplegar varios contenedores en un mismo equipo.
  • Velocidad rápida al arrancar los servicios.
  • Contenedores muy livianos haciendo más fácil el almacenaje, transporte y despliegue.
  • Los contenedores se pueden desplegar en muchas plataformas.
  • Aplicaciones libres de las dependencias que tenga, solamente tendrán que estar instaladas en la máquina anfitrión.


COMANDOS BÁSICOS


  • DESCARGAR UNA IMAGEN (POR EJEMPLO DEBIAN)

    • EJM: docker pull debian


  • LISTA DE IMÁGENES DISPONIBLES EN EL SISTEMA

    • EJM: docker images


  • LISTA DE CONTENEDORES DISPONIBLES EN EL SISTEMA

    • EJM: docker ps -a

    La opción -a sirve para ver los contenedores que están en ejecución y los que no.


  • PARA VER IMÁGENES (POR EJEMPLO UBUNTU)

    • EJM: docker search ubuntu


  • EJECUTAR IMAGEN POR PRIMERA VEZ (POR EJEMPLO UBUNTU)

    • EJM: docker run -it ubuntu

    Solo para la primera vez que vayamos a ajecutar ya que sino abrirá un nuevo contenedor, en vez de abrir el que creamos anteriormente.


  • PARA ABRIR LA IMAGEN CREADA ANTERIORMENTE

    • EJM: docker start 'id o nombre'


  • PARA EJECUTAR LA IMAGEN

    • EJM: docker exec -it 'id o nombre' bash


  • PARAR CONTENEDOR

    • EJM: docker stop 'id o nombre'


  • ELIMINAR CONTENEDOR

    • EJM: docker rm 'id o nombre'


  • RENOMBRAR CONTENEDOR

    • EJM: docker rename 'id o nombre' 'nombre dado por nosotros'


  • FORMA DE VISUALIZAR DATOS DEL DOCKER

    • EJM: docker ps -a --format="ID:\t{{.ID}}\nName:\t{{.Names}}"

    EJM: Usamos el --format y después añadimos filtros según lo que queramos visualizar.
